## Field Contract

Future implementation must treat file identity as immutable metadata derived
from read-only observation, not as a license to transform original materials.

| Field | Required meaning | Phase 1 rule |
|---|---|---|
| `source_uri` | Absolute `file://` URI for a user-approved source file or bounded source path. | Define only; do not open, scan, or normalize real raw content. |
| `sha256` | Hex SHA-256 fingerprint of exact file bytes. | Define only; Phase 1 does not hash real raw files. |
| `file_size` | Byte length observed at fingerprint time. | Define only; Phase 1 does not stat real raw files. |
| `mtime` | Source filesystem modification time observed at fingerprint time. | Define only; later drift checks must not rewrite files. |
| `first_seen_at` | UTC timestamp when IDS first records the source identity. | Define only; must be generated by future manifest logic from actual scan time. |

## Idempotency And Duplicate Rules

Future Phase 2/3 behavior must be deterministic:

- same `source_uri` and same `sha256` means the same source identity;
- same filename but different `sha256` is a conflict or version candidate,
  not a silent overwrite;
- same `sha256` with different `source_uri` is duplicate content with distinct
  provenance, not a reason to delete either original;
- repeated scans must not create duplicate documents, chunks, jobs, manifest
  rows, evidence facts, or audit log events;
- unsupported or unreadable files must receive explicit status in later phases
  and must not be silently skipped;
- no duplicate rule may move, delete, overwrite, compact, or normalize original
  files in place.

## Future Status Contract

Phase 1 defines status names that later phases may implement:

| State | Meaning | Required behavior |
|---|---|---|
| `ORIGINAL_RAW_READY` | Source identity can be observed under read-only and bounded rules. | Allow later metadata scan only; do not mutate source. |
| `ORIGINAL_RAW_NOT_CONFIGURED` | No approved raw source path was supplied. | Fail closed and request configuration. |
| `ORIGINAL_RAW_PATH_BLOCKED` | Source path is outside allowed roots, unsafe, missing, or unvalidated. | Block scan before side effects. |
| `ORIGINAL_RAW_READ_BLOCKED` | File cannot be read by a future authorized scanner. | Record explicit failure status; do not skip silently. |
| `ORIGINAL_RAW_HASH_CONFLICT` | Same name or source identity maps to different hash evidence. | Record conflict; do not overwrite existing identity. |
| `ORIGINAL_RAW_DUPLICATE_CONTENT` | Same hash appears at multiple source URIs. | Preserve provenance; do not delete originals. |
| `ORIGINAL_RAW_MANIFEST_UNSAFE` | Manifest target would store raw payload, secrets, unbounded data, or unsafe path. | Block manifest write before side effects. |
| `ORIGINAL_RAW_UNKNOWN` | State cannot be classified from available evidence. | Fail closed; do not scan, import, index, or report. |
