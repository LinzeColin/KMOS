## 固定运行序列

1. 晚间 automation 运行 official collector，落盘 `TEMPORARY_REMINDER`，发送保持关闭。
2. 后续晨间 automation 先查找最新 canonical evening reminder 且尚无 final 的已完成工作日。
3. 如存在待核对日期，必须先取得独立官方导出原件，并按企业实际全部列完成逐员工逐格对账。
4. 无独立原件时固定 `OFFICIAL_EVIDENCE_MISSING`；存在任何人员、列或单元格差异时固定 `OFFICIAL_FINAL_RECONCILIATION_BLOCKED`，且不得运行 DWS collector 生成伪官方结论。
5. 只有独立逐格证据 PASS 且后续 DWS 内部门禁也通过，才可写 canonical `final` raw/manifest 和 aggregate-only `one_page.md`。
6. 新月累计只读取 canonical `final` archive；所有 legacy 和暂时提醒 archive 仅保留审计。
7. 晨间 automation 再生成当天 `TEMPORARY_REMINDER`，发送仍关闭。

- DWS 授权和 browser policy READY；未授权则 fail closed。
- `notification_status=NOT_SENT_OWNER_DISABLED`。
- 不执行 target probe、resend 或 sender。
- 不把 synthetic fixture 作为真实工作日证据。
- 私有 raw、报告正文、一页结果和 manifest 只留在私有归档，不提交 GitHub。
- GitHub 只保存脱敏聚合一页结果，不保存员工明细、官方原件、私有路径或逐格差异正文。
