# KMFA 前端交互与人类可读报告规范 v1.1

## 1. 风格

1. 全中文。
2. 商务蓝色系。
3. KM标识。
4. 大面积使用白、浅蓝、深蓝、灰阶。
5. 警示色只用于小徽标、状态点、标签，不影响阅读。
6. 不使用噱头式文案。

## 2. 首页模块

```text
经营总览
项目成本
回款应收
财务资金
开票纳税
数据源检查
待处理事项
报告中心
```

## 3. 数据源检查板

列头固定：

```text
来源系统 / 业务板块 / 文件包 / 公司主体 / 银行或系统账户 / 账户或报表 / 频率 / 状态 / 影响报告 / 处理规则 / 下一步
```

## 4. 报告结构

经营报告只展示管理者需要阅读的内容：

```text
本期经营摘要
项目成本与毛利
财务资金与利润质量
回款应收与账龄
开票纳税与政策证据
口径差异摘要
下期重点事项
```

数据源检查、内部日志、处理队列属于系统功能，不塞进经营报告正文。

## 5. 前端写入限制

前端允许写：

```text
字段映射规则
项目匹配确认
差异处理意见
人工确认记录
备注
报告发布审批
```

前端禁止写：

```text
原始上传文件
原始hash
原始抽取值
已发布历史报告
已批准历史处理事件
```
