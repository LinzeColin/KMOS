# KMFA HTML / UIUX / 报告样板强制继承规范 v1.2

## 1. 目的

防止 Codex 只读 Roadmap 文本而遗漏系统首页、报告样式、数据源检查板、人工处理工作台等 UIUX 验收标准。

## 2. 继承要求

- 系统必须全中文。
- 不得出现明显 AI 痕迹、机器人感、智能体感、营销话术感。
- 使用蓝色系商务风；黄色仅限少量警示标签，不作为大面积底色。
- 页面必须像真实经营管理软件，不像演示 PPT。
- 信息密度高但可读，先给经营结论，再给证据、口径、处理事项。
- 数据检查、差异处理、报告等级必须可见。
- 前端操作必须写入控制事件、处理意见、映射规则、审批记录，不得污染原始上传数据。

## 3. 开发必须读取的 HTML

见 `20_HTML_UIUX_报告预览/00_HTML总入口_KMFA_v1_2.html`。
