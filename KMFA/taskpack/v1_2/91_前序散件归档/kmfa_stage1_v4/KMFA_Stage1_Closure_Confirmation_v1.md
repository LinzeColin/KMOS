## 3. D15 MVP必须包含

1. 文件导入：支持zip、xlsx、xls/WPS识别、csv、pdf附件登记。
2. metadata：每次导入写入source、file_hash、schema_status、operator、timestamp。
3. 数据源检查板：五色状态；缺关键数据自动降级或阻塞。
4. 项目成本分析：项目成本归集、成本完整性、利润质量、异常成本、人工复核队列。
5. 经营摘要报告：经营结果、项目组合、项目成本、现金回款、税务政策摘要、下期事项。
6. 输出形式：系统Dashboard、HTML报告、PDF报告、必要CSV/Excel下载。
7. 通知提醒：报告ready或重大风险时发邮件提醒，不发送完整报告正文。

## 4. D15明确不做

- 不做红圈/金蝶/WPS自动登录抓取。
- 不接真实账号密码明文。
- 不自动报税、自动付款、自动对外发送报告。
- 不做合同全文OCR和自动法律判断。
- 不做完整报价漏项检测和合同风险扫描，只保留后续接口。
- 不把原始敏感数据提交公开GitHub。

## 5. D20稳定准确门禁

| 门禁 | 验收标准 |
|---|---|
| 准确性 | 选定样本项目关键数字与人工表核对；差异必须有原因和证据 |
| 可追溯 | 每个关键数字能追到文件、工作表、字段、导入批次、公式 |
| 稳定性 | 连续3次导入同一数据，报告关键结果一致 |
| 异常处理 | 缺文件、坏文件、WPS/OLE、字段缺失都进入明确错误状态 |
| 报告质量 | 报告无AI痕迹、全中文、表格图表清晰、非专业人员可读 |
| 治理 | 人工复核、行动项、报告版本、通知日志全部写入metadata |
