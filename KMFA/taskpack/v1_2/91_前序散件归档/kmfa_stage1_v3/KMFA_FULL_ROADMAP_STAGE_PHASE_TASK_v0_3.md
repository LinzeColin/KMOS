# KMFA 经营分析系统｜Codex 全程开发 Roadmap v0.3

状态：ChatGPT 阶段一讨论稿，用于阶段二研究后继续修订。  
项目路径：`CodexProject/KMFA`  
开发形态：独立项目；后期以入口、报告中心索引、运行状态 manifest 方式接入 OpMe。  
核心角色：经营负责人、财务负责人、税务政策负责人、董事层。系统菜单保留经营总览、财务资金、税务政策、数据接入、报告中心、系统治理。  
行业范围：建工建安、机电机加工、水泥钢铁能源化工委外安装维修，核心为项目制工程经营。

---

## 全局原则

1. 给 Codex 的所有任务使用 **Stage → Phase → Task** 三层结构。
2. 每个 Stage 不超过 5 个 Phase，每个 Phase 不超过 5 个 Task。
3. 每次 Codex run 只处理一个项目、一个 Stage/Phase/Task、一个 Acceptance。
4. 先 READ-ONLY PLAN，再实现。
5. 原始敏感数据不进入公开 GitHub。
6. `KMFA/metadata` 只保存 manifest、hash、schema、状态、证据索引、报告索引和通知记录。
7. 报告不能暴露明显生成痕迹，必须像真实经营软件输出。
8. 内部口径转换与差异核对机制不作为报告显性标题；报告只保留“口径差异与管理调整摘要”。
9. 数据源检查板属于系统功能，不属于正式经营报告正文。
10. 未来 OpMe 集成不得污染 KMFA MVP，也不得污染 OpMe 现有工程运维模块。
