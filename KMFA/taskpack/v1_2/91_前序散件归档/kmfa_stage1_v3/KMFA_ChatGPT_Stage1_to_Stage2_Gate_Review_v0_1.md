## 4. 阶段二应该做什么

阶段二不写正式代码，重点输出研究与任务包材料：

### 4.1 官方文档核验

1. 红圈：导出能力、API/JSAPI/RPA 可行性、字段结构、权限边界。
2. 金蝶：不同产品线 API/导出报表、科目余额、凭证、应收应付、报表格式。
3. WPS：表格/轻维表/云文档 API、共享目录、权限与导出。
4. 税务/数电票：发票导出、字段、税率、进项销项、认证抵扣相关数据字段。
5. GitHub：metadata、secrets、敏感数据治理、未来 CI 检查。
6. OpMe：未来入口接入方式与隔离原则。

### 4.2 行业模型深研

1. 建工建安收入、成本、合同履约、签证、结算、质保金、回款特点。
2. 委外安装维修项目的真实毛利模型。
3. 机电机加工项目成本结构。
4. 水泥、钢铁、能源、化工客户项目交付与回款风险。
5. 项目制企业的现金预测和应收催收优先级模型。

### 4.3 税务政策深研

1. 科技型中小企业评价。
2. 高新技术企业认定。
3. 研发费用加计扣除。
4. 专精特新与小巨人政策。
5. 建筑服务、加工维修、安装服务、委外服务可能涉及的税率、进项抵扣、简易计税等口径。

### 4.4 报告和系统模板完善

1. 周报、月报、季报、半年报、年报模板。
2. 经营总览、财务资金、税务政策三套主视图。
3. 数据源检查板最终字段。
4. 报告 PDF 版式标准。
5. 用户可读性验收标准。
6. 信息质量密度验收标准。

### 4.5 TaskPack 准备

2. KMFA_REQUIREMENTS_MATRIX.md
3. KMFA_DATA_SOURCE_CONTRACT.md
4. KMFA_REPORT_SPEC.md

## 5. 阶段二需要你提供的材料

阶段二可以先不提供真实敏感数据，但如果你能提供脱敏样本，研究质量会明显提高。

### A. 最有价值的脱敏样本

| 来源 | 建议提供 | 脱敏要求 |
|---|---|---|
| 红圈 | 经营数据、合同数据、回款数据、财务数据导出样例 | 客户名、项目名、金额可替换，保留字段结构 |
| 金蝶 | 科目余额表、凭证明细、应收应付、利润表/现金流导出样例 | 金额可缩放，摘要可脱敏，保留科目与列名 |
| WPS | 客户信息、账期、应收账龄、发票台账、报价表样例 | 客户、联系人、电话、账号脱敏 |
| 银行 | 单个主体单个账户流水样例 | 账号、户名、金额可脱敏，保留日期、摘要、收支方向 |
| 合同 | 合同字段清单或脱敏合同摘要 | 不要发合同全文，先发字段结构 |
| 政策材料 | 研发项目台账、人员工时、知识产权清单样例 | 人名和证书号脱敏 |

### B. 不要提供

```text
账号密码明文
完整银行账号
客户敏感信息
合同全文
工资明细
税务申报原文
未脱敏发票全文
```

## 6. 阶段二 Stop Condition

遇到以下情况，应停止进入更深层实现：

1. 需要真实账号密码明文才能继续。
2. 需要把敏感原始数据提交到公开 GitHub。
3. 用户要求系统自动申报、自动付款或自动改财务系统。
4. 数据源字段完全未知且没有脱敏样本。
5. 官方文档无法确认接口能力，却要直接承诺自动接入。
6. 报告模板无法区分经营管理视图和正式财务税务口径。

---

**我建议可以进入 ChatGPT 阶段二。**
