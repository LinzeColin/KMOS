# KMFA 零差异验证与测试计划 v1.1

## 1. 零差异定义

项目成本分析中，系统输出与A0权威基准的金额字段必须分毫不差。任意0.01元差异为失败。

## 2. 校验对象

1. 合同额/收入。
2. 支出合计。
3. 成本分类。
4. 毛利。
5. 毛利率。
6. 回款/开票/结算相关字段。
7. 汇总行。
8. 报告引用单元格。

## 3. 必测场景

| 测试 | 要求 |
|---|---|
| 金额格式 | 支持千分位、负数、括号负数、元、万元 |
| 异常值 | 空白、横杠、井号不得默认为0 |
| PDF与Excel冲突 | 保留两边源值，进入差异队列 |
| 同源多引用 | 发现不一致则重跑链路 |
| 重复导入 | 同一批数据连续三次结果一致 |
| 坏文件 | 不崩溃，进入失败状态 |
| 缺字段 | 不静默跳过，进入部分/阻塞状态 |

## 4. 压力与极限测试

1. 100个项目、1000个成本行、10000个交易行。
2. 同名项目、多主体、多账户、多客户别名。
3. 高精度小数、负数、红冲、退款、保证金。
4. 大量待处理差异并发重跑。
5. 报告生成失败回滚。

## 5. 通过标准

1. 0.01元差异全部被发现。
2. 无float金额。
3. 每个报告数字有lineage。
4. 未处理差异能阻断A级报告。
5. 原始文件hash不变。
