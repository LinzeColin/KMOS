# KMFA 给 Codex 的开发 Roadmap｜18 个 Stage｜v1.2

本文件是给 Codex 的开发任务拆分与路线规划。它不是 ChatGPT 阶段划分；当前对话属于 ChatGPT 阶段三，交付物是 TaskPack 与本 Roadmap。



## v1.2 补漏门禁：HTML 与报告样板必须进入开发路线

v1.1 缺少 HTML 文件，本版 Roadmap 明确补充：

```text
任何涉及 S10 报告生成、S11 前端基础界面、S12 人工处理工作台、S18 验收的任务，必须读取 20_HTML_UIUX_报告预览/ 下的 HTML 样板。
```

验收时必须输出：

```text
1. 已读取HTML样板清单
2. 页面/报告与样板差异说明
3. 数据源检查板状态色实现截图或HTML输出
4. 项目成本专题报告HTML输出
5. 人工处理工作台HTML/前端输出
6. 报告中心HTML/前端输出
```

HTML 缺失、未读取或未体现在实现中，S10-S12、S18 不得通过。

## 总规则

- Codex 开发 Roadmap 共 18 个 Stage，符合 10–30 个 Stage 的要求。
- 每个 Stage 不超过 5 个 Phase；本版每个 Stage 为 3 个 Phase。
- 每个 Phase 不超过 5 个 Task；本版每个 Phase 为 3 个 Task。
- 真实开发时间仅作资源安排参考，不是必须用满的耗时。质量门禁通过可以提前交付；质量门禁未通过不得交付。
- 任何业务金额不得使用浮点数；必须使用整数分或 Decimal。
- 任意 0.01 元差异必须进入差异报告，不允许静默通过。
- 前端只能写控制事件、处理意见、映射规则、审批记录；不能改写原始上传文件、原始hash、原始抽取值。
