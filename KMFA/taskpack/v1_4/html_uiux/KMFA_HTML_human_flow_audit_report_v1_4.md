# KMFA HTML/UIUX 人类流程核验报告 v1.4

## 1. 结论

本轮对 v1.4 修复版 HTML 样板执行本地 Playwright 静态人类流程点击核验。

```text
HTML 文件数：6
控制项/链接/输入核验行数：54
PASS：54
WARN：0
FAIL：0
```

结论：v1.4 样板达到 `FAIL=0`。Codex 后续实现必须以 v1.4 样板为 UIUX 交互基线。

## 2. v1.2 问题

v1.2 交付存在以下问题：

```text
1. HTML 文件进入包内，但没有被设为强制验收基线。
2. 部分页面按钮没有真实可见反馈。
3. 搜索、状态切换、处理意见、影响预览、重跑链路没有被系统化验收。
4. UI 仍有文档堆叠感，缺少真实用户操作路径。
```

## 3. v1.4 修复

v1.4 补入：

```text
1. HTML 总入口。
2. 系统全流程可点击样板。
3. 经营分析报告可点击预览。
4. 数据源检查板可点击预览。
5. 待处理事项工作台可点击预览。
6. Codex开发任务控制台可点击预览。
7. 自动化点击审计脚本。
8. 自动化点击审计结果。
```

## 4. Codex UIUX 验收要求

Codex 不得只做“看起来像页面”的前端。必须满足：

```text
按钮有动作。
搜索有筛选。
状态可变更。
状态变更写入控制事件，不污染原始数据。
人工处理有意见、有影响预览、有重跑链路。
报告有章节切换、附表导出、打印/另存流程。
数据源检查板能按来源系统、业务板块、文件包、公司主体、银行账户等维度呈现。
```
