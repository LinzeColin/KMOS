# 功能清单

## 当前功能

- Codex 受控钉钉 DWS 全文件原始数据归档。
- 运行前中文 preflight：分别列出 `auto` 群与 `manual_only` 群、常驻/项目类型、cursor 起点和风险。
- 每群独立 cursor，成功到 `hasMore=false` 后才更新。
- 项目群成功增量扫描后，如连续 7 天无新增消息和文件，自动记录最后一次有消息的时间为完工时间，并转为 `manual_only`。
- 长跨度常驻群/项目群按全时、全盘、全量、深度完整收集处理，不截页、不缩短时间窗、不抽样。
- 运行 heartbeat：长扫描按页、按资源批次刷新 `reports/current_run_heartbeat.json`，供 automation 判断是否仍在推进。
- 失败资源 3 次 backoff，超过后 `exhausted=1`，默认不再每日主动重试。
- `manual_only` 群不主动扫描，但会从历史 manifest/messages 重建当前包目录和质量证据。
- 最近 60 天热存储输出到 `/Users/linzezhang/onedrive/DWS_Outputs.zip`。
- 60 天及以上文件本体迁入 `/Users/linzezhang/onedrive/DWS_Archive/<群名>/files/MM/<文件>`。
- 历史冷存储 `files/MMDD/` 会在真实运行中迁移为 `files/MM/`。
- 冷存储不使用年份目录，不生成 zip。
- SQLite manifest 永久保留冷热路径、hash、大小、失败尝试和 cursor 证据。
- Downloads 构建目录验证后删除。
- 中文人类可读报告与机器 JSON 附录。
- skill/config 指纹同步 Notion，失败时写 pending。
- 目标群增删改时必须同步检查 Codex automation；验证从当前配置动态读取所有 enabled 群。

## 明确不做

- 不做业务周报、月报、经营分析结论。
- 不使用本机 launchd 无人值守自动运行。
- 不生成每群 `<群名>_latest.zip`。
- 不删除 SQLite manifest、冷存储文件、必要 reports 或 Notion 同步状态。
