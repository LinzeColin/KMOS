# 08_自动验收规则

## 1. 自动验收目标

自动验收不是给用户看的形式文件，而是 Agent 内部必须执行的质量门禁。未通过时不得交付正式 Excel/PDF。

## 2. 必须检查项

| 类别 | 检查项 | 失败处理 |
|---|---|---|
| 文件命名 | 是否为 `经营管理分析报表 YYYYMM.xlsx`、`董事会经营分析摘要 YYYYMM.pdf` | 失败 |
| 输入文件 | 6 个业务输入包是否齐全 | 失败 |
| Sheet 顺序 | 可见 Sheet 是否严格等于 v6 顺序 | 失败 |
| Sheet 标签颜色 | 是否与 v6 基准一致 | 失败 |
| 禁用词 | 是否出现 Series、责任角色、CxO/CEO/CFO/COO/CMO/CTO、源文件状态等 | 失败 |
| 乱码 | 是否出现替换字符或常见编码乱码 | 失败 |
| 公式错误 | 是否出现 `#REF!`、`#DIV/0!`、`#VALUE!`、`#NAME?`、`#N/A` | 失败 |
| 百分比 | 涉及比例是否两位小数百分比 | 失败或警告，严格模式下失败 |
| 资金汇总 | 资金总流入、资金总流出、累计资金净流量是否为公式 | 失败 |
| 第 30 行 | `04_2026年资金汇总` 第 30 行是否为累计资金净流量 | 失败 |
| 图表 | 必需图表是否存在，标题是否正确 | 失败 |
| 可读性 | 列宽行高是否存在明显截断风险 | 严格模式下失败 |
| PDF | PDF 是否存在、文件大小是否合理、期间是否一致 | 失败 |

## 3. 人工复核项

自动脚本通过后，Agent 仍必须人工/视觉复核：

- 首页 KPI 是否完整显示。
- 图表是否遮挡表格。
- PDF 是否分页合理。
- 颜色是否与 v6 基准一致。
- Sheet02/03/04 源表复刻是否破坏原格式。
- 经营判断是否可执行，不空泛。

## 4. 失败即停原则

任何红线失败，Agent 必须停止交付正式文件。不得向用户说“已完成”但实际存在失败项。

## 5. 验收脚本

使用：`scripts/validate_deliverables.py`。

脚本输出 JSON，包含：

- `status`: passed / failed
- `errors`: 错误项
- `warnings`: 警告项
- `checks`: 每项检查结果
- `metadata`: 期间、文件、Sheet、图表、时间戳
