# 05_PDF董事会摘要规范

## 1. 输出命名

`董事会经营分析摘要 YYYYMM.pdf`

## 2. PDF 内容结构

PDF 必须面向非财务专业读者，强调经营判断、关键数据、风险和建议行为。默认 3–5 页。

| 页码 | 内容 |
|---:|---|
| 1 | 首页总览转 PDF：核心 KPI、月度回款与开票趋势、风险机会行动摘要 |
| 2 | 回款、开票、应收、项目闭环：开票未回款、完工未开票、合同未完工、重点应收 |
| 3 | 资金与保证金：资金流入、资金流出、累计净流量、保证金未退、可回收现金池 |
| 4 | 客户与行业：行业结构、客户贡献、客户质量、回款/开票贡献 |
| 5 | 资质人力与趋势预测：人员缺口、产能约束、未来13周回款预测、口径风险 |

如果内容不足 5 页，可压缩为 3 页；不得超过 8 页。

## 3. PDF 语言规范

- 全中文。
- 不出现 CxO/CEO/CFO/COO/CMO/CTO。
- 不出现“责任角色”。
- 不出现“AI 生成”“机器维护”“源文件状态”。
- 对非专业读者解释指标含义，但不要堆公式。

## 4. 数据口径

PDF 中所有数字必须来自同期间 Excel。不得在 PDF 中新增 Excel 没有的关键金额。

必须在 PDF 页脚或末页写：

`数据口径：本摘要取自《经营管理分析报表 YYYYMM.xlsx》，金额默认单位为万元，源表复刻页按源表单位显示。`

## 5. 视觉规范

- 横向 A4 或 16:9 页面均可，必须保证可读。
- 标题层级清晰，字号足够。
- 图表清晰，图例中文。
- 不得出现内容截断、乱码、图表挤压。
- 不得把 Excel 中未渲染完整的截图直接贴入 PDF。

## 6. 董事会摘要必须回答的问题

1. 本期现金是否改善？
2. 回款与开票是否匹配？
3. 哪些应收最影响现金？
4. 项目交付是否转化为开票与回款？
5. 保证金是否存在可释放现金？
6. 资金净流量是否有压力？
7. 未来 13 周回款趋势如何？
8. 数据口径有哪些需要注意的差异？
9. 下期最可行的管理动作是什么？

## 7. 禁止事项

- 不得用旧版 PDF 改文件名冒充本期 PDF。
- 不得用没有通过验收的 Excel 导出 PDF。
- 不得在 PDF 中写无法追溯的经营结论。
- 不得把风险写成笼统口号，必须绑定关键数据和建议行为。
