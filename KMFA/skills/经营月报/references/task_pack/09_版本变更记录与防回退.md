# 09_版本变更记录与防回退

## 1. 当前基准

当前基准版本为 v6。后续所有月度/周度输出均必须基于 v6，不得回退到 v1–v5 的结构、颜色、语言或图表方式。

## 2. 历史关键要求沉淀

| 版本问题 | 当前 v6 规则 |
|---|---|
| 图表出现 Series1/Series2 | 必须使用中文业务系列名 |
| 图表不可移动不可编辑 | 必须使用 Excel 原生可编辑图表 |
| 首页字体同质化 | 标题、KPI、说明必须有字号层级 |
| 行高列宽不足，出现 `#####` | 生成后必须自动调整并验收 |
| Sheet 顺序混乱 | 固定 00–10 顺序 |
| 机器页污染前台 | 机器维护信息仅可隐藏或并入 Sheet10 |
| 明细污染管理报表 | 主 Excel 不放完整明细 |
| 资金汇总公式被写成值 | 资金总流入/流出/累计净流量必须公式 |
| 口径被重新发挥 | 使用本任务包的指标口径与跨表耦合规则 |

## 3. 防回退机制

Agent 在执行任何修改前必须：

1. 读取 `golden/经营管理分析报表 v6基准.xlsx`。
2. 读取 `config/v6_spec.json`。
3. 对比本次输出与 v6 基准的 Sheet 顺序、标签颜色、必需图表和禁用词。
4. 只允许更新数据、公式、图表数据源、PDF文字和必要的本期口径说明。
5. 不允许重做整体设计。

## 4. 新需求处理

如果用户提出新需求：

- 先判断是否会破坏 v6 基准结构。
- 若只是新增分析，必须放在 Sheet10 后方或隐藏辅助页；不得插入前面打乱顺序，除非用户明确要求。
- 若改变源表复刻页，必须先确认不会破坏源表格式。
- 若改变命名规则，必须先确认是否覆盖 `FileName YYYYMM`。

## 5. 版本记录模板

每次执行后，在内部日志记录：

| 字段 | 内容 |
|---|---|
| period | YYYYMM |
| baseline | v6 |
| input_files | 本期输入文件 |
| output_excel | 输出 Excel |
| output_pdf | 输出 PDF |
| validation_status | 通过/失败 |
| changes | 与上期相比的真实数据变化 |
| known_limitations | 源数据缺失、格式异常、口径差异 |

## 6. 长期治理建议

当连续 3 个月输出稳定后，可以再做：

- 第二个 Excel：`明细表 YYYYMM.xlsx`。
- 历史快照库。
- 客户行业映射表。
- 项目状态映射表。
- 自动生成董事会行动追踪表。

以上新增内容不能破坏当前 v6 主报告。
