# 04_Excel生成规范

## 1. 执行流程

Agent 必须遵循：

1. 读取 v6 基准文件，禁止空白重建样式。
2. 读取本期 6 个业务输入包。
3. 建立输入映射和字段映射。
4. 刷新数据表、公式和图表。
5. 调整行高列宽，确保所有内容完整显示。
6. 扫描禁用词、乱码、`#####`、公式错误。
7. 运行自动验收脚本。
8. 只在全部通过后输出正式 Excel。

## 2. 不得做的事

- 不得新增用户未要求的可见 Sheet。
- 不得删除 v6 已验收结构。
- 不得把公式改成值。
- 不得把图表变成图片。
- 不得把明细混入管理报表。
- 不得为了通过校验手工改平差异。
- 不得输出测试数据、虚拟客户、虚拟金额。
- 不得使用英文默认图例。
- 不得出现 `#####` 或内容被截断。

## 3. 数据写入原则

| 类型 | 规则 |
|---|---|
| 源表复刻页 | 尽量保持源表结构、格式、行列顺序、公式和单位 |
| 管理汇总页 | 使用清洗后的真实数据，金额统一万元 |
| 比率 | 两位小数百分比 |
| 资金合计 | 必须为公式 |
| 跨表差异 | 保留并解释，不得手工抹平 |
| 缺失值 | 显示 `--` 或空白，并在口径说明中解释；不得填 0 伪装 |

## 4. 布局和可读性

交付前必须确保：

- 所有列宽足够显示标题和关键数据。
- 所有行高足够显示三行 KPI、长标题和建议行为。
- 所有可见单元格水平居中、垂直居中、自动换行。
- 首页 KPI 卡片中金额和比例字号大于说明文字。
- 表头颜色、字体、边框与 v6 基准一致。
- Sheet 标签颜色与 v6 基准一致。
- 图表不覆盖表格，不压缩得不可读。
- 图例在底部或右侧，且为中文业务对象。

## 5. 公式规范

公式必须可复制、可追溯、可审计。禁止把关键合计写成值。

关键公式必须至少包括：

- 资金总流入。
- 资金总流出。
- 当月净流量。
- 累计资金净流量。
- 开票未回款率。
- 回款/开票比。
- 未退比例。
- 占比贡献率。
- 同比、环比。

所有除法公式必须使用 `IFERROR` 或等价逻辑，分母为 0 时显示 `--` 或空值，不得出现 `#DIV/0!`。

## 6. 明细处理

当前主 Excel 是“经营管理分析报表”，不放完整明细。后续如用户授权生成第二个 Excel，命名为：

`明细表 YYYYMM.xlsx`

明细表才承接所有源文件明细和开票纳税资金汇总表中的全部明细表。

## 7. PDF 导出前检查

Excel 通过验收后，再导出 PDF。PDF 不得从旧版本导出，必须对应同一期间、同一 Excel。
