> 用途：把“当月原始发放 Excel”整理成“模板文件 / 上月最终正确版”的结构、样式、公式和 Sheet 顺序，并在交付前完成金额、归属、公式、格式、文件完整性和 ABExcel 校验。

把“当月原始文件”整理成“模板文件 / 上月最终正确版”的格式和样子。

2个来源齐全，5个维度一致，2检验，12类检查，ABExcel通过，20项测试通过，才算是准确无误能交付验收

## 5. 金额绝对一致规则

### 5.1 金额底层规则

1. 金额必须使用十进制口径读取，不得用二进制浮点数作为最终判断依据。
2. 所有金额统一转换为整数分字段：`amount_cents = round_exact(amount * 100)`。
3. 原始金额超过两位小数，必须标记为异常，不允许静默四舍五入。
4. 显示可以为两位小数，但最终校验必须比较 `*_cents`。
5. 金额差异通过标准为：

```text
source_cents = final_cents = oracle_cents
cents_delta = 0
```

6. 不接受“总额差异小于 0.01 元”作为通过标准。
7. 不允许为了让汇总一致而修改明细金额。
8. 不允许只看汇总，不查明细。

### 5.2 金额字段最少覆盖

| 字段 | 标准字段名 | 校验规则 |
|---|---|---|
| 实发金额 | `net_pay_cents` | 原始 = 最终 = 汇总 = Oracle |
| 主体公司支付 | `main_company_pay_cents` | 原始 = 最终 = 分表小计 = 汇总 = Oracle |
| B公司支付 | `b_company_pay_cents` | 原始 = 最终 = 分区小计 = 汇总 = Oracle |
| 报销/发票金额 | `reimburse_or_invoice_cents` | 原始 = 最终 = 分区小计 = 汇总 = Oracle |
| 扣借支 | `advance_deduction_cents` | 原始 = 最终 = 汇总 = Oracle |
| 提前发 | `next_month_advance_cents` | 原始 = 最终 = 汇总 = Oracle |

### 5.3 独立 Oracle 校验

必须从“当月原始文件 + 本模板规则”独立重建 B 校验账本。B 校验账本不得从最终发放文件复制单元格值。

```text
A = 最终发放版抽取结果
B = 原始数据 + 规则独立重建结果
通过 = A 与 B 在行、字段、金额分、路由、公式、格式指纹上全部一致
```

## 7. B公司/报销与金额列路由规则（最高优先级）

本节是最高优先级业务规则。Agent 必须先看金额列是否有数字，再用“B公司/报销”字段决定归属。不得只看字段文本，也不得用社保公司覆盖“B公司/报销”的含义。

### 7.1 基础定义

| 名称 | 定义 |
|---|---|
| 主体公司 | 该行用于主体公司支付归属的公司。若原始文件字段名为“社保公司”，也按该行主体支付公司处理。 |
| 公司名称 | 能从当月原始文件公司字段、模板分区名、或用户明确给出的公司清单中识别出的公司。 |
| 有数字 | 金额列非空，且可解析为数字。金额为 0 时保留原值参与校验，但不得凭空新增其他金额。 |
| 公司名称/报销 | 公司名称与“报销”之间有 `/` 或 `／`，例如 `曦悦/报销`。 |
| 公司名称报销 | 公司名称与“报销”直接相连，中间没有 `/` 或 `／`，例如 `曦悦报销`。 |

必须区分 `公司名称/报销` 和 `公司名称报销`。两者不能互相改写，不能按同一种规则处理。

### 7.2 字段解析顺序

解析“B公司/报销”字段时，按以下顺序执行：

1. 清理前后空格、全角空格、不可见字符。
2. 保留 `/`、`／`，不得删除斜杠。
3. 优先匹配 `公司名称/报销`。
4. 再匹配 `公司名称报销`。
5. 再匹配纯 `报销`。
6. 再匹配纯 `公司名称`。
7. 无法识别时，标记为差异 / 无法确认项，不允许猜测。
8. 不允许内置人员姓名、历史月份人员、历史金额作为路由依据。

### 7.3 主体公司支付列

如果 `主体公司支付列` 有数字，则该金额归该行 `主体公司`。

如果 `主体公司支付列` 无数字，则不生成主体公司支付金额。

`B公司/报销` 字段不改变主体公司支付列的归属。

### 7.4 B公司支付列

如果 `B公司支付列` 有数字，且 `B公司/报销 = 公司名称`，则 B公司支付金额归该 `公司名称`。

如果 `B公司支付列` 有数字，且 `B公司/报销 = 公司名称/报销`，则 B公司支付金额归该 `公司名称`。

如果 `B公司支付列` 有数字，且 `B公司/报销 = 公司名称报销`，则 B公司支付金额归该 `公司名称`。

如果 `B公司支付列` 有数字，但 `B公司/报销` 为空、纯 `报销`、或无法识别公司名称，则标记为差异 / 无法确认项。

如果 `B公司支付列` 无数字，则不生成 B公司支付金额。

### 7.5 发票金额列

如果 `发票金额列` 有数字，按下表决定发票金额归属：

| B公司/报销字段 | B公司支付列 | 发票金额列 | 发票金额归属 |
|---|---|---|---|
| 空白 | 任意 | 有数字 | 差异 / 无法确认 |
| 公司名称 | 任意 | 有数字 | 差异 / 无法确认 |
| 报销 | 任意 | 有数字 | 主体公司 |
| 公司名称/报销 | 有数字 | 有数字 | 主体公司 |
| 公司名称/报销 | 无数字 | 有数字 | 该公司名称 |
| 公司名称报销 | 任意 | 有数字 | 该公司名称 |
| 无法识别文本 | 任意 | 有数字 | 差异 / 无法确认 |

如果 `发票金额列` 无数字，则不生成发票金额。

### 7.6 组合场景

如果 `主体公司 = 武汉`，且 `B公司/报销 = 曦悦`，且 `主体公司支付列` 有数字，且 `B公司支付列` 有数字，则：

- 主体公司支付金额归 `武汉`；
- B公司支付金额归 `曦悦`；
- 若发票金额列无数字，则不生成发票金额。

如果 `主体公司 = 武汉`，且 `B公司/报销 = 曦悦/报销`，且 `主体公司支付列` 有数字，且 `B公司支付列` 有数字，且 `发票金额列` 有数字，则：

- 主体公司支付金额归 `武汉`；
- B公司支付金额归 `曦悦`；
- 发票金额归 `武汉`。

如果 `主体公司 = 武汉`，且 `B公司/报销 = 曦悦/报销`，且 `主体公司支付列` 有数字，且 `B公司支付列` 无数字，且 `发票金额列` 有数字，则：

- 主体公司支付金额归 `武汉`；
- 发票金额归 `曦悦`。

如果 `主体公司 = 武汉`，且 `B公司/报销 = 曦悦报销`，且 `主体公司支付列` 有数字，且 `发票金额列` 有数字，则：

- 主体公司支付金额归 `武汉`；
- 发票金额归 `曦悦`；
- 如果 B公司支付列也有数字，则 B公司支付金额同时归 `曦悦`。

### 7.7 Agent 执行伪代码

```text
main_target = none
b_pay_target = none
invoice_target = none

if main_company_pay_has_number:
    main_target = main_company

if b_company_pay_has_number:
    if field_form in [公司名称, 公司名称/报销, 公司名称报销]:

    else:
        error = 差异/无法确认

if invoice_amount_has_number:
    if field_form == 报销:
        invoice_target = main_company
    elif field_form == 公司名称/报销:

            invoice_target = main_company
        else:
            invoice_target = parsed_company
    elif field_form == 公司名称报销:
        invoice_target = parsed_company
    else:
        error = 差异/无法确认
```

### 7.8 路由验收标准

| 检查项 | 通过标准 |
|---|---|
| 主体公司支付 | 有数字即归主体公司，差异 = 0 |
| B公司支付 | 有数字且可识别公司时归对应 B 公司，差异 = 0 |
| 发票金额 | 严格按 7.5 表格归属，差异 = 0 |
| 未定义组合 | 必须进入差异报告，不允许猜测 |
| 分表归属 | 总额一致但分表归属不同，仍判定不通过 |

## 8. 分区落表规则

同一原始行可以同时产生多类金额归属。每一列独立落表，不能互相覆盖。

| 金额列 | 落表目标 | 规则 |
|---|---|---|
| 主体公司支付列 | 主体公司支付分区 | 归该行主体公司 |
| B公司支付列 | B公司支付分区 | 归 7.4 识别出的公司 |
| 发票金额列 | 发票/报销分区 | 归 7.5 计算出的主体公司或 B 公司 |

禁止为了总额一致而移动分区。分表小计、汇总页金额、Oracle 账本必须逐行、逐金额分、逐归属回勾一致。

## 10. 公式规则

1. 模板中原本是公式的位置，最终必须仍是公式。
2. 分表小计、分表合计、汇总页合计必须使用公式。
3. 行数变化后，公式范围必须同步调整到实际明细范围。
4. 不允许把公式写死成数值。
5. 不允许出现以下错误：

```text
#REF!
#VALUE!
#DIV/0!
#NAME?
#N/A
#NULL!
#NUM!
```

6. 如果工具无法可靠计算 Excel 公式缓存，必须用独立 Python/脚本账本重算金额，并在 ABExcel 中保留公式差异和公式错误检查。

## 11. 生成最终发放版

1. 以模板文件为基底复制。
2. 清除模板旧月份明细，不清除公式框架、样式框架、分区结构。
3. 写入当月标准化账本数据。
4. 按业务归属写入对应分表/分区。
5. 调整公式范围。
6. 复制新增行样式。
7. 设置有效区域水平居中、垂直居中。
8. 保存为：


## 12. ABExcel 校验检查文件规范

每次都必须生成：

```text
{目标月份}-{YYYYMMDD}_校验ABExcel检查.xlsx
```

### 12.1 A/B 定义

| 名称 | 含义 |
|---|---|
| A 文件 | 待检查最终发放版 |
| B 文件 | 由原始数据 + 模板规则独立重建出来的校验版，或用户指定的参考正确版 |
| ABExcel 检查 | 把 A/B 同名 Sheet 复制到校验工作簿中，检查值、公式、错误、核心金额和格式指纹 |

如果任务是比较两个用户提供的文件：

| 名称 | 含义 |
|---|---|
| A 文件 | 版本 1 |
| B 文件 | 版本 2 |
| 正确性判断 | 必须回到当月原始文件，按业务规则判断谁正确 |

### 12.2 ABExcel 必须包含的 Sheet

| Sheet 名称 | 内容 |
|---|---|
| AB_控制台 | 每个 Sheet 的差异数量和总通过状态 |
| AB_核心金额回勾 | 实发、主体、B公司、报销/发票、扣借支、提前发回勾 |
| AB_差异清单 | 所有差异单元格、A值、B值、差异类型、是否修复 |
| AB_格式指纹 | 字体、字号、水平居中、垂直居中、数字格式、边框、填充、合并信息 |
| A_汇总、A_各分表 | A 文件各 Sheet 副本 |
| B_汇总、B_各分表 | B 文件各 Sheet 副本 |

### 12.3 AB_控制台列结构

| 列 | 字段 |
|---|---|
| A | Sheet 名 |
| B | 比对范围 |
| C | 值差异数 |
| D | 公式差异数 |
| E | 公式错误数 |
| F | 核心金额差异数 |
| G | 格式指纹差异数 |
| H | 行数差异数 |
| I | 结论 |

### 12.4 值差异数量公式（金额按分比较）

在 `AB_控制台!C2` 放入并向下填充：

```excel
=LET(s,$A2,r,$B2,a,IFERROR(INDIRECT("'A_"&s&"'!"&r),""),b,IFERROR(INDIRECT("'B_"&s&"'!"&r),""),d,IF(ISNUMBER(a)*ISNUMBER(b),ROUND(a*100,0)<>ROUND(b*100,0),NOT(EXACT(a&"",b&""))),SUM(--d))
```

说明：金额按整数分比较；非金额文本按完全一致比较。

### 12.5 公式差异数量公式

在 `AB_控制台!D2` 放入并向下填充：

```excel
=LET(s,$A2,r,$B2,ra,INDIRECT("'A_"&s&"'!"&r),sr,ROW(INDEX(ra,1,1)),sc,COLUMN(INDEX(ra,1,1)),rows,ROWS(ra),cols,COLUMNS(ra),SUM(MAKEARRAY(rows,cols,LAMBDA(i,j,--NOT(EXACT(IFERROR(FORMULATEXT(INDIRECT("'A_"&s&"'!"&ADDRESS(sr+i-1,sc+j-1,4))),""),IFERROR(FORMULATEXT(INDIRECT("'B_"&s&"'!"&ADDRESS(sr+i-1,sc+j-1,4))),"")))))))
```

### 12.6 公式错误数量公式

在 `AB_控制台!E2` 放入并向下填充：

```excel
=LET(s,$A2,r,$B2,x,INDIRECT("'A_"&s&"'!"&r),y,INDIRECT("'B_"&s&"'!"&r),SUM(--ISERROR(x))+SUM(--ISERROR(y)))
```

### 12.7 总结论公式

在 `AB_控制台!I2` 放入并向下填充：

```excel
=IF(AND(C2=0,D2=0,E2=0,F2=0,G2=0,H2=0),"通过","不通过")
```

总结果单元格：

```excel
=IF(COUNTIF(I:I,"不通过")=0,"ABExcel检查通过","ABExcel检查不通过")
```

### 12.8 格式指纹

普通 Excel 函数不能完整读取字体、字号、边框、填充色、垂直居中和合并单元格，因此 Agent 必须用脚本生成格式指纹。

每个有效单元格格式指纹至少包含：

```text
font_name | font_size | bold | italic | font_color | fill_color | border_left | border_right | border_top | border_bottom | num_format | horizontal | vertical | wrap_text | merged_range
```

格式指纹差异数必须为 0。

## 15. 负向测试 Mutation Test

必须验证校验器真的能抓错。做法：复制最终发放文件，制造一个小错误，重新跑校验，校验器必须失败。

最低负向测试：

| 测试 | 故意改动 | 期望 |
|---|---|---|
| MT-001 | 任意金额 +0.01 | 金额差异失败 |
| MT-002 | 把一个可识别路由文本改成无法识别文本 | 路由差异失败 |
| MT-003 | 删除最终文件一条有效明细 | source_only_rows 失败 |
| MT-004 | 复制最终文件一条有效明细 | duplicate 或 final_only 失败 |
| MT-005 | 把小计公式改成数值 | formula_missing 失败 |
| MT-006 | 把金额改成文本 | type_check 失败 |
| MT-007 | 修改有效区域字体或对齐 | style_fingerprint 失败 |

如果故意改错后校验器没有失败，则校验体系无效，禁止声称通过。

## 20. 禁止事项

1. 不允许跳过原始数据逐行检查。
2. 不允许只做版式，不做金额校验。
3. 不允许只看汇总，不查明细。
4. 不允许把公式写死成数值。
5. 不允许沿用模板旧人员、旧金额、旧备注。
6. 不允许把 `公司名称/报销` 和 `公司名称报销` 混为同一种规则。

12. 不允许不生成 ABExcel 检查文件。
13. 不允许未通过 20 项测试仍声称最终正确。
14. 不允许发现差异却不输出 MD 和 Excel 差异报告。
15. 不允许金额使用 float 作为最终判断依据。
16. 不允许原始金额超过两位小数时静默四舍五入。
17. 不允许把文本金额当作正常金额通过。
18. 不允许在无真实测试日志或真实校验文件时声称“通过”。
19. 不允许内置历史人员姓名、历史月份个案、历史金额作为规则。
20. 不允许输出文件名缺少发给 Agent 的当天日期。
