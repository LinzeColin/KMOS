# Dingtalk-routine-check｜功能清单

## 只新增一个 automation/skill

名称：`Dingtalk-routine-check`

中文名：`钉钉工作检查`

这个 automation 不负责创建或管理 DWS 采集。它只读取已有输入目录：

```text
/Users/linzezhang/Library/CloudStorage/OneDrive-Personal/DWS_Outputs/
  付款请示群/
  生产管理群/
```

## 功能 1：读取 DWS 输出

- 读取两个群的 `manifest.csv`。
- 读取两个群的 `chat_records.csv/jsonl`。
- 读取已下载图片/文件路径。
- 统一 message、file、sha256、sender、time、group。
- 不重新扫描钉钉。
- 不下载新图片。

## 功能 2：固定工作检查

- 每日、每周、每月第三周周五规则检查。
- 支持单人发送规则。
- 支持多人任选其一规则。
- 支持图片、文件、文本消息。
- 到截止时间仍未发，提醒张霖泽。
- 迟发、低置信度、发错群、疑似发错表，提醒张霖泽。

## 功能 3：资金账户明细表 OCR 监控

- 仅监控付款请示群、杨婷。
- 识别 `资金账户明细表`。
- 提取总可用现金。
- 低于硬阈值 50 万：P0 红色。
- 低于软阈值 100 万：P1 黄色。
- 正常：P2 绿色，只记录不打扰。

## 功能 4：资金流水明细 OCR 结构化

- `资金流水明细` 和 `资金明细` 视为同一类。
- 与 `资金账户明细表` 独立检查。
- 提取日期、事由、收付款人、类别、收入、支出、银行、备注。
- 目前不做大额支出异常。
- 目前不接已审批未付款金额。

## 功能 5：日志与数据库

- 不生成汇总 Excel。
- 写 SQLite 私有 ledger。
- 写 OneDrive JSONL 日志。
- 写 notification log。
- 写 OCR job/extraction log。
- 写数据质量问题日志。
- 后续 KPI/工作数据分析可复用这些结构化事实。

## 功能 6：本机存储与清理

- 活跃 SQLite 放本机 private_runtime，避免 OneDrive 锁表。
- JSONL、DB 备份、OCR 缓存、cleanup report 放 OneDrive。
- 支持 cleanup dry-run/apply。
- 不删除 DWS 原始输出目录。

## 功能 7：GitHub 实时同步治理

- 所有 skill/code/metadata/tests 改动必须在 main 分支同步。
- 不开 branch。
- 不开 PR。
- push 前跑 validator/tests。
- 失败则不 push。
- 提供 `git_autosync.py`，供 Codex Desktop automation 调用。
