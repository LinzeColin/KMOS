# 商业交付级 UI/UX 重构验收记录（2026-07-20）

## 为什么有这个单元

Owner 指令：完成剩余任务后「重构 UIUX 触感交互用户体验操作系统流程，确保能商业交付」。
此前的 App 是功能完备但工程师审美的原型：十二个页签横排一行、样式全部内联、
无加载/失败态设计、无移动端适配、无暗色主题——能用，不能当产品交付。

## 改了什么

**零业务逻辑变更**。所有接口调用、数据处理、决策/重跑/导出流程、append-only 纪律
与诚实边界文案一字未动；改的只是呈现层与信息架构。

1. **设计系统**（新增 `src/app.css`）：颜色/间距/圆角/阴影全走 CSS 令牌；
   明暗两套主题由 `prefers-color-scheme` 自动切换，图表用 echarts 内置 dark 主题跟随；
   金额与计数一律 `tabular-nums` 等宽数字、右对齐。
2. **信息架构**：十二页签从「一行横排」改为**分组侧边栏**——
   总览／数据底座／经营专题／处置与追溯／交付与运维 五组，每项配无文本 `svg` 图标；
   置顶栏显示当前页名 + 质量/报告/GO 三徽章（`NO_GO` 红色风险样式）+ 数据截至批次。
3. **触感与交互**：卡片悬浮升起、按钮按压缩放、行悬停变色、页签切换渐入、
   键盘方向键在页签间移动（roving tabindex）、`focus-visible` 焦点环全覆盖、
   `prefers-reduced-motion` 尊重减动效设置。
4. **状态设计**：加载=骨架屏（不再是一行「加载中…」）；接口失败=显式报错卡
   （写明「页面不猜数据，取不到就明说」）；空表=居中空态。
5. **导航注意点**：数据说有事，入口就亮点——我在哪（有卡住的事）、差异工作台
   （有 open 断言）、排程健康（有失败=红点/空跑=黄点）。不用逐页翻。
6. **状态胶囊**：open/closed、成功/失败、已开/空跑、风险等级等一律语义色胶囊，
   替代裸色文字。
7. **移动端**：≤920px 侧边栏折成横向滚动页签条，指标卡与表格自适应；
   竖屏 375px 实测可读可点。

## E2E 兼容是硬约束（逐条守住）

- 页签仍为 `button role=tab`，`textContent` 与页名**严格相等**（图标是无文本 `svg`）；
- `aria-selected` 翻转逻辑不变；页眉三元组仍在唯一 `header` 元素内；
- 影响重跑页第一个 `select` 仍是资产选择器；
- 决策理由/重跑理由占位符、「闭案 → closed」按钮文案、「已追加事件」提示、
  「链完整：是」「旧版本全保留：是」等断言文本逐字保留。

## 实测（真起服务、真开页面、真点全流）

镜像 `kmfa-app:uiux` 以 CI 同法构建（`npm run build` 后 `dist` 烧入镜像），容器起于本机。

**十二页逐页点验**（浏览器真点，读 `aria-selected` 与面板内容）：
12/12 全部正确选中、内容渲染、零页面错误、无卡死骨架屏、无加载失败。

**全流端到端**（与 CI 完全同版本 playwright 1.49 容器、同脚本、干净状态卷）：

```
PASS  [登录] 页眉三元组: 质量 Q3 / 报告 D / Q4 / D / NO_GO / 3-9-2-1
PASS  [首页] 含 BLK-001 / S18 / 版本 0.1.4
PASS  [检查板] 覆盖矩阵含真实资产数
PASS  [成本页] 阻塞如实呈现且无编造毛利
PASS  [差异处理] 三选一决策真写入（MANEVT-APP-…）
PASS  [影响预览] goods_movement 17,764 由血缘边算出
PASS  [重跑] 四层链 4/4、链完整、旧版本全保留
PASS  [报告导出] HTML/CSV/PDF 三格式真下载、魔数与水印全对
== 10/10 PASS ==
```

**移动端**：375×812 实测——横滚页签条 + 注意点、徽章竖排、表格换行可读。
**暗色**：桌面暗色实测——排程健康页胶囊/结论条、数据管线页 echarts 柱图
（kingdee_ledger 162,979 行等真实数据）全部适配。

## 不含糊的边界

- **服务器上还没生效**：要等下一次 Redeploy。线上现跑旧界面，功能不受影响。
- 置顶栏原用毛玻璃（`backdrop-filter`），实测在无障碍截图管道下诱发合成撕裂帧，
  且低端设备渲染贵——已换实底色，视觉几乎无差。此为本单元唯一一次返工。
- 本单元不改后端、不改数据、不改判定口径；`NO_GO` 红徽章只是把既有结论显示得更醒目，
  不代表任何业务状态变化。
