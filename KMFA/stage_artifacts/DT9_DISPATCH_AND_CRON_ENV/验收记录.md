# 考勤消息发不出去 — 三处修复记录（2026-07-20）

Owner 反馈「一直没收到钉钉」。基建线（OVH Singapore VPS-1）从基础设施一路查到业务逻辑，报了 5 个问题、修掉 4 个（均为运行容器内热修，重部署即失效），第 5 个归我。
本次在仓里修 **3 处**——1 处是基建线点名归我的，2 处是他们热修但没进仓的源头。

## 一、先自己验证，再动手

基建线的每条结论我都自查过，其中**有一条他们的怀疑不成立**：

| 他们的判断 | 我的核实 |
|---|---|
| `dispatch_reports_to_targets()` 在遍历前早退 | **属实** |
| 怀疑 `TEMPORARY_REMINDER` 走了「不发送」分支 | **不成立**——`grep REMINDER_RUN_TYPES` 全仓只两处用到：`collection_integrity.py` 做路由、`notification_targets.py:240` **只用来选失败标签**，没有任何「reminder 不发送」分支 |
| 真因 | 空 `targets` 一路掉到函数末尾，被 `_summarize_dispatch_status([])` 归成裸 `FAILED` |

实证：
```
>>> _summarize_dispatch_status([])

```
这正是他们看到的症状：`notification_status: FAILED`、`target_results: []`、`messages: []`、**无 `failure_reason`**。排查的人只能看着一个 FAILED 猜。

> 顺带核过一个我自己的误判：`_summarize_dispatch_status([{'status':'SENT'}])` 也返回 `FAILED`——但那是我随手构造的字典形状不对。`_target_send_result` 实际吐的是 `management_status`/`hr_status`，**这条不是 bug**，没有据此改代码。

## 二、修复①：派发早退必须说清原因（归我的那条）

三处显式化，凡「没发出去」的收据一律带非空 `failure_reason`：

| 场景 | 新状态 | reason 内容 |
|---|---|---|
| 零目标 | `NOT_SENT_NO_TARGET_SELECTED` | 文件路径 + `total=N` + `filter=all` |
| 目标全禁用 | `NOT_SENT_ALL_TARGETS_DISABLED` | 逐个列出 label |
| 走完循环仍无结果 | `NOT_SENT_NO_DELIVERABLE_TARGET` | 启用目标数 + 提示查 `reports`/`resolved_channel` |

**不再退化成裸 `FAILED`。**

## 三、修复②：cron 不继承容器环境（基建线热修，进仓固化）

`cron.d` **不做变量展开**，所以 entrypoint 用**实际值**渲染环境头，不能照抄 `${VAR:-0}` 字面量。每行都是实测出来的坑：

- `HOME=/root` —— 缺了 dws 找不到 `/root/.dws`，首跑直接 `DWS_AUTH_REQUIRED` rc=2
- `TZ` —— 缺了 cron 按 UTC 评估排程，晨报在错误时刻触发（绕开 #100/#108 的锚定）
- `KMFA_DELIVERY_ENABLED` —— 缺了任务按 dry-run 跑，ledger 记 `delivery_enabled:"0"`，消息永远发不出去

并加自检：`HOME` / `TZ` / `KMFA_DELIVERY_ENABLED` / `KMFA_ATTENDANCE_ARCHIVE_ROOT` 少一行就拒绝启动。

## 四、修复③：报告归档根写死了 Owner 的 Mac 路径

`ONEDRIVE_ROOT = "/Users/linzezhang/OneDrive/dingtalk_attendance"` 硬编码。容器里被 `mkdir` 成一个假目录，报告写进去，**容器一重建全部丢失，且没有任何报错**——又一次静默失效。

改为 `os.environ.get("KMFA_ATTENDANCE_ARCHIVE_ROOT", <原 Mac 路径>)`：本机默认不变（保 parity），云端由 entrypoint 覆盖成 `/var/lib/kmfa/attendance` 并 `mkdir -p`。

实测：
```
默认（本机 parity）: /Users/linzezhang/OneDrive/dingtalk_attendance
覆盖后（云端）    : /var/lib/kmfa/attendance
```
