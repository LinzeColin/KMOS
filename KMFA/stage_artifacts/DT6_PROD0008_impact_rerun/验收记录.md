## 四、真开页面逮到的 2 个问题

**① 下游影响面少报断言域与报告。** 页面上「受影响断言域」「受影响报告」都是 `—`，但 `expense_lines`/`tax_composition` 明明是税费轴断言的取数源。查断言表发现 `our_source` 的真实长相带限定：
```
_staging.expense_lines(6403) via _staging.tax_composition
_staging.kingdee_ledger（book=武汉开明）
```
我用的是精确相等匹配，这些**全都漏掉**。改为按表名出现即命中后，同一资产带出 `material、tax` 两个域 + 3 份报告。契约测试补交叉校验：逐条按表名子串重算一遍，结果须与接口一致。

**② 轮次号在同一秒内会撞车。** 原实现 `RERUN-APP-{秒级时间戳}-{资产哈希}`，Owner 手快双击就会撞出同一个 id，**两轮留痕被并成一轮**，账记错。改为按已有轮次递增（不依赖挂钟），时间戳留在 `rerun_at` 字段。契约测试连发两轮断言轮次号不同。
