# 钉钉考勤安全联锁演练记录（2026-07-18）

考勤是唯一在产、最敏感的技能（SKL.0003 双跑一刀切换）；其 dws 守卫（`dws_auth_guard.py`：环境闸 `KMFA_DINGTALK_ATTENDANCE_ALLOW_DWS_COMMANDS=1` + `~/.dws/pat_policy.json` 策略文件双闸）此前从未在容器实测。本单两态演练：

| 场 | 状态 | 结果 |
|---|---|---|
| A | 无任何闸直接跑 `attendance-morning` | **rc=2**，结构化输出 `status: DWS_AUTH_REQUIRED`（含时区/无样本数据等自检字段）——拒绝先于任何 `dws` 触碰 ✅ |
| B | 只开环境闸、缺 `pat_policy.json` | **同拒 rc=2**——单闸不放行，安全链必须完整 ✅ |

## 语义确认

- rc=2 会进 `run_skill` 台账并触发崩溃告警层（有 webhook 时）——**认证缺失的考勤运行本就该告警**，行为正确。
- 实例日顺序自然成立：`dws auth login --device`（手机确认）→ 基座写入 `pat_policy.json`（openBrowser=false）→ 双闸齐 → 考勤 dry-run 双跑开始。
- 至此在产链的三道安全面全部实测：投递闸（`KMFA_DELIVERY_ENABLED=0` 双跑纪律）、认证闸（本单双闸）、告警分层（`DT9_SKL0002_runskill_drill`）。
