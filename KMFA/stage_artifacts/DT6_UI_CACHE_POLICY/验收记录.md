# 前端缓存策略验收记录（2026-07-20 夜）

## 事发

`#159` 合并后 Coolify 部署 `qjmaae4avesuqelbdddlzo3o` 于 09:24Z `finished`
（deploy 工作流轮询确认「部署成功」），服务器已在跑新界面——但 Owner 刷新
`kmfa.linzezhang.com` 看到的仍是旧界面（原话「前端还是没有变 一片稀烂」）。
根因：入口 `index.html` 无任何 `Cache-Control`，浏览器按启发式缓存继续用
旧 html，旧 html 引用旧哈希资产——**服务器换了新、用户看着没变**。

## 改动（`app/main.py` 三处响应头，其余零变更）

- `/ui/` 与 `/`（入口 html）：`Cache-Control: no-cache, must-revalidate`
  ——浏览器每次必回源验证，部署换新立即可见；
- `/ui/assets/*`（带内容哈希的资产）：`Cache-Control: public, max-age=31536000, immutable`
  ——内容变则文件名必变，可放心永久缓存，反而更快。

这是 `SPA` 生产部署的标准缓存策略；从此「已部署但用户看不见」这类问题结构性消失。
