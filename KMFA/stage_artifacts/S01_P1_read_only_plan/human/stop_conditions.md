# KMFA S01-P1 Stop Conditions

更新时间: 2026-06-29

## 1. 本轮停止条件

遇到以下情况立即停止，不继续实现:

1. 需要创建正式 `KMFA/` 项目骨架。
2. 需要写 UI、报告、业务逻辑或导入器。
3. 需要接红圈、金蝶、WPS、银行、税务等自动接口。
4. 需要复制、提交或展示原始敏感经营数据。
5. 需要修改原始下载包或原始上传文件。
6. 需要把参考脚本当作正式项目工具使用。
7. 需要跨越一个以上 Phase。

## 2. 后续开发全局停止条件

任何 Stage/Phase 中发现以下情况，立即停止并先修:

1. 原始数据、原始 hash、原始抽取值被污染。
2. 业务金额使用 `float`。
3. 任意 0.01 元差异被静默通过。
4. 差异未处理但报告等级升级为 A。
5. 关键报告数字缺失 lineage。
6. 前端直接改写 raw 层或已发布历史报告。
7. 缺少 Stage/Phase/Task 验收证据。
8. 公开 repo 中出现账号密码、token、原始财务明细、银行流水、合同、工资、税务申报明文。
9. 缺少回滚方案或无法解释数据来源。

## 3. GitHub 上传停止条件

Stage 内任一 Phase 未完成、Stage 复审未完成、复审问题未修复、或存在敏感明文风险时，不上传 GitHub。
