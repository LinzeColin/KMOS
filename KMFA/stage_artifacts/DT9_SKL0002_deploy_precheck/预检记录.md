## 排掉的雷

| # | 雷 | 表现 | 修复 |
|---|---|---|---|
| 1 | `apt` 装不存在的 `flock` 包 | 构建第一步即失败（bookworm 中 `flock` 属基础镜像自带的 `util-linux`） | 从 apt 清单删去，构建尾部加 `command -v flock` 断言 |
| 2 | `dws` release 资产名全猜错 | 三个候选 URL 均 404，安装脚本中止 | 实测 v1.0.52 资产名为 `dws-<os>-<arch>.tar.gz`，列首位 |
| 3 | `dws` 锁文件空（首次信任） | 首装记录任意哈希，防篡改无效 | 从官方 `checksums.txt` 预填 `dws.sha256.lock`（linux-arm64/amd64、darwin-arm64 三行），实例日校验非信任 |
| 4 | `rapidocr`→OpenCV 缺 `libGL.so.1` | OCR 引擎导入即崩（slim 镜像通病） | apt 增 `libgl1 libglib2.0-0` |

另修一处可用性问题：`entrypoint.sh` 不透传参数（任何 `docker run <img> <cmd>` 都被吞进前台 `cron -f` 挂死）→ 带参时 `exec "$@"`；`docker-compose.yml` 的 `env_file` 改 `required: false`（缺 secrets 时容器可起、按既有设计告警空跑，便于预检）。
