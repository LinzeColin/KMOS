# DT8 健康周检落地记录（2026-07-18）

roadmap DT8 的「备份与自动化健康周检」此前只有日备份；本单补齐**周检**：`self-audit` 任务（周日 03:00 悉尼）自动复跑三件固化验证——证据链（`evidence_check`）、血缘时效（`lineage stale`）、双平面四项目门（含语义门/中文门）。rc≠0 时走 `run_skill` 崩溃告警层。

## 容器演练（两轮，抓获一真雷）

1. 首轮 **rc=1**：双平面门的渲染一致检查**原地重渲染**，只读挂载的仓库上直接 `OSError: Read-only file system`——正是周检设计要抓的那类环境级断裂。
2. 修法演进：影子浅克隆（败于 worktree 的 `.git` 指针文件）→ **tar 影子**（排除 `.git`/数据对象/node_modules/私有运行时，解包到 `/tmp` 跑，跑完即删）。
3. 终轮 **rc=0 全绿**：`PASS 30 条证据链`、`血缘 FRESH`、`双平面 4 项目 PASS`，台账 JSON 行完整。

## 语义

- 周检覆盖=公开面完整性（证据/血缘/双平面）；**私有库类门禁**（`gate_runner` 三门、匹配复现）需 DuckDB，实例上有库时可手动 `run_skill.sh` 扩展，不入默认周检（容器无私有库为常态）。
- 与日备份、双跑台账共同构成 DT8「贯穿工作流」的运维底座。
