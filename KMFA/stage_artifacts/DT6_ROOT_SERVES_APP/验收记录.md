# 根路径直达应用验收记录（2026-07-20 深夜）

## 事发与误诊更正（不粉饰）

Owner 两次反馈「前端还是没变 一片稀烂」并贴出截图。截图显示：裸域名
`kmfa.linzezhang.com` 打开的是**一张早期静态摘要页**（对账断言 19/30、
staging 数据行、断言清单表——后端 `app/static/index.html`，前 React 时代的产物），
真正的 App 一直挂在 `/ui/` 路径。

**此前两轮诊断都不对**：既不是「没部署」（部署确实 `finished`），也主要不是
「浏览器缓存」（上一单元 `DT6_UI_CACHE_POLICY` 的归因不全）——根因是
**根路径本身就服务着一张过时页面**。用户不该需要知道 `/ui/` 才能用产品。
截图同时也证明了：服务器活着、真实数据在出（263,758 行、19/30 断言、
页眉三徽章 质量 Q3／报告 D／NO_GO 俱在）——错的是「根路径给用户看什么」。

## 改动

- `main.py` 根路径：`307` 重定向到 `/ui/`——裸域名直达应用本体；
- **删除** `app/static/index.html`（连同 `STATIC` 常量）——那张静态页永久退场，
  不留「还能被谁看到」的可能；
- `test_smoke.py` 根路径测试改为断言 `307 → /ui/` 且跟随后落在 App
  （防止将来有人把静态页加回来）。

## 实测（真起容器、真发请求）

```
GET /（不跟随）→ HTTP/1.1 307 Temporary Redirect ; location: /ui/
GET /（跟随） → <title>KMFA 经营分析</title>，页面含 App 挂载点 root
pytest 84 passed（python:3.12 容器）
```

`.app` 启动器本就打开 `/ui/`（`launcher.sh` 第 14 行），不受影响。

  重新评——若仍不满意，按其具体意见继续改，不拿本次修复当挡箭牌。
