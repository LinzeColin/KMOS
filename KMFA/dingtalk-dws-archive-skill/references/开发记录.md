## 2026-07-07 项目群自动完工规则

目标：

- 项目群在 Codex 受控增量扫描成功到 `hasMore=false` 后，如果本轮无新增消息、无新增文件信号，且最后一次有消息的时间距当前已满 7 天，自动视为完工。
- 自动完工时写回 `config/target_groups.yaml`：`scan_mode: "manual_only"`、`completed_at`、`completed_date`、`last_active_message_time`、`completion_reason`、`completion_run_id`、`auto_completed: true`。
- `completed_at` 和 `last_active_message_time` 使用最后一次有消息的时间，不使用 automation 运行时间。

验收标准：

- 未成功到扫描完成边界时不更新 cursor，也不自动完工。
- 项目群只要本轮有新增消息或新增文件信号，就不自动完工。
- 自动完工后的项目群不会被 automation 主动扫描，但允许用户显式指定手动补扫、重建包和查历史。

## 2026-07-08 automation 运行失败修复

根因：

- 最新 automation 失败在 DWS `discovery/jsonrpc` 调用阶段，错误为 `dial tcp ... i/o timeout`，尚未进入群消息扫描。
- 当前 DWS doctor 可登录且网络可达，但延迟约 5 秒，说明问题是瞬态网络/服务调用韧性不足，不是配置群缺失。
- 每日 automation 的 cwd 一度变成 `/Users/linzezhang/CodexProject`，不符合本项目固定目录。

修复：

- DWS CLI 调用统一传 `--timeout`，默认 `dws_http_timeout_seconds=120`。
- 对 `io_timeout`、`request_timeout`、`dial tcp`、`context deadline` 等瞬态错误最多重试 3 次，退避基准 5 秒。
- 更新每日 automation cwd 回本项目固定目录，并在 prompt 中要求检查 DWS doctor/timeout/retry 证据。
- `manual_only` 群不再从当前交付包消失；真实运行会从历史 manifest/messages 重建跳过群目录和质量证据。
- 历史冷存储 `files/MMDD/` 目录会迁移为当前标准 `files/MM/`，并同步更新 SQLite manifest 的 cold 路径。

验收标准：

- DWS doctor 通过，允许有版本更新 warning。
- 单群 `--probe` 能完成 group resolve/message scan。
- 当前包包含所有 enabled 群目录，包括 manual_only 群的 manifest/status/chat_records。
- 冷存储根目录无 `files/MMDD/`、年份目录或 zip。
- Python 编译、shell wrapper、plist、输出结构验证通过。
