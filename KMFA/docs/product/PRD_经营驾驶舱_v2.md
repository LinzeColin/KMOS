# PRD：KMFA 经营驾驶舱 v2（推倒重构）

2026-07-20 · 状态：已实施（本 PR）· Owner 指令原文：
「你这全部都是开发和后端看的 根本不是财务、管理、公司、用户看的 自己组建一个
PRD软开UIUX设计架构组织架构客户经理团队全部推倒重构软件的UIUX交互触感反馈用户体验信息第一性」

## 一、团队与方法

虚拟团队四角色（财务负责人／老板·管理层／资深 B 端 UIUX 设计师／客户经理·客户成功）
独立出需求与批判（存档 `团队需求纪要.json`），产品与架构由主线收敛。四方结论高度一致：

1. **首屏答错了问题**：治理状态表+路线图回答「系统建得怎么样」，用户问的是
   「我的钱和账怎么样、哪些要我拍板」。
2. **界面说的不是中文**：`断言`、`analyzed_open`、`sha256`、`rc=0`、`staging` 行数
   ——每个词都在收「翻译税」。
3. **轻重不分**：天天用的拍板和一年不点的 `数据管线` 并排 12 个一级入口。
4. **坏消息只甩代号不给出路**：`Q4/D/NO_GO` 裸奔；诚实是原则，但诚实≠让用户猜有多糟。

## 二、产品原则（信息第一性）

- **先答问题，再给证据**：每页第一行是人话结论；工程细节降灰、折叠、下沉。
- **金额是第一语言**：先说差多少钱，行数/哈希/代号一律脚注化。
- **坏消息三件套**：照实说（不许美化 `NO_GO`）＋缺什么＋谁来做/何时好。
- **拍板即回执**：动作 200ms 内可见反馈，待办计数即时变化，留痕可查。
- **诚实边界不动摇**：不发明数据、不隐藏水印、不借改版升报告等级——
  改的是「怎么说」，不改「说什么」。

## 三、信息架构（12 页签 → 8 页 5 组）

| 组 | 页 | 内容（全部来自既有接口，后端零改动） |
|---|---|---|
| 总览 | **今天** | 人话判决横幅（含出路）＋等我拍板队列（按金额倒序，点进直达）＋钱三卡＋数据新鲜度一行＋报告出口卡 |
| 钱 | **回款与账龄** | 原账龄回款，术语人话化（已对平/待拍板 胶囊） |
| 钱 | **开票与税务** | 原开票纳税，红线表述人话化 |
| 钱 | **项目成本** | 原项目成本；「算不出金额」横幅带缺什么/谁能解 |
| 拍板 | **待拍板** | 原差异工作台；待处理排最前，三选一拍板留痕不变 |
| 报告 | **报告下载** | 原报告中心；水印状态明示「内部版·不可对外」 |
| 后台 | **数据底账** | 源检查板＋数据管线＋重新核算（原影响重跑）＋操作留痕（原审计日志）四节合一 |
| 后台 | **系统自检** | 排程健康（主体）＋自动化任务清单（原技能）＋治理详情折叠（原我在哪） |

砍掉的一级入口：`我在哪`（治理内容下沉系统自检折叠区）、`影响重跑`（并入数据底账
「重新核算」）、`审计日志`（并入数据底账「操作留痕」）、`源检查板`/`数据管线`/`技能`
（并入后台两页）。路线图不再占用户屏幕。

## 四、术语表（界面用左列，右列仅在 `code` 小字/悬停出现）

| 人话（界面） | 机器词（降级保留） |
|---|---|
| 核对项 / 核对规则 | `断言` |
| 已查明·待拍板 | `analyzed_open` |
| 已对平 | `closed_matched` / `closed` |
| 已排除 | `excluded` |
| 暂不可对外——关键账未对平，报告仅限内部 | `Q4 / D / NO_GO` |
| 已接入原始记录 N 条（脚注） | `staging` 数据行 |
| 数的来龙去脉 | 血缘节点/边 |
| 只记不改、可回查 | `append-only` |
| 防篡改校验码 | `sha256` |
| 昨夜自动核对成功/失败 | `rc=0` / `rc≠0` |

保留不译（语义即业务动作、且有留痕契约）：三选一拍板按钮文本（如「闭案 → closed」）
——按钮文案与审批台账/接口字段一字对应，翻译显示会造成留痕与所见不一致。

## 五、触感与反馈

- 判决横幅常驻琥珀/绿色，不弹窗轰炸、不感叹号恐吓、不折叠藏丑。
- 金额等宽数字、千分位、右对齐；账龄超期色阶递进。
- 拍板成功即回显事件号与哈希前缀；重新核算完成回显「四层已重算、旧版本全保留」。
- 下载按钮点击即真得文件；水印与「可否外发」写在卡面。
- 键盘方向键换页、焦点环、减动效偏好、移动端横滚页签（底部固定栏列为 v2.1 后续）。

## 六、非目标（本次不做）

- 不改任何后端接口与数据判定口径；不新增数据能力。
- 不做角色权限分屏（单 Owner 场景）；不做底部移动导航（列入 v2.1）。
- 不改变 G 门禁、水印、`append-only` 等治理机制——只改呈现。

## 七、验收（与 E2E 同步）

八步真流程（打开→今天→回款与账龄→项目成本→待拍板真写决策→数据底账真跑四层重算
→报告三格式真下载→系统自检）全绿入 CI；页眉三徽章保留；成本页永不出现「毛利率」。
