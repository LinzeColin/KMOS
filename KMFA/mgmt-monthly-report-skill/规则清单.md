# 规则清单

## 输入规则

1. 每期必须先确认 `YYYYMM`。
2. 输入必须通过 7 个槽位 manifest 显式映射，不得把目录里所有 Excel 自动纳入计算。
3. `.xls` 不能直接进入正式流程，必须转换为 `.xlsx` 后重新校验。
4. 任一必需槽位缺失、冲突或无法识别时，不得生成正式 Excel/PDF。
5. `invoice_tax_cash` 的开票、销售回款、资金汇总内容只能来自对应源表或明确同义 sheet。
6. `project_status_contracts` 必须记录项目状态和主合同信息来源；不能只靠合同登记猜测完工或未开工。

## 输出规则

1. 正式 Excel 文件名必须是 `经营管理分析报表 YYYYMM.xlsx`。
2. 正式 PDF 文件名必须是 `董事会经营分析摘要 YYYYMM.pdf`。
3. 正式 Excel 必须基于 v6 结构，不得空白重做样式。
4. 可见 Sheet 必须严格保持 00-10 顺序。
5. 不得新增、删除、重命名、重排可见 Sheet。
6. 辅助 Sheet 如确需存在，必须隐藏且名称以 `_` 开头。
7. 图表必须是 Excel 原生可编辑图表，不得截图替代。
8. PDF 必须从同期间、已验收通过的 Excel 取数。

## 口径规则

1. 金额计算不得使用 float 作为审计依据；脚本与公式应使用 Decimal、整数分或 Excel 公式。
2. 百分比统一两位小数。
3. 缺失值显示 `--` 或空白，不得填 0 伪装完整。
4. 跨表差异必须进入 `09_经营口径核对`。
5. 不得为了通过校验手工抹平差异。
6. 资金总流入、资金总流出、累计资金净流量必须为公式。

## GitHub 和本机保留规则

1. `KMFA/mgmt-monthly-report-skill/` 存 skill、规则、脚本、模板和测试。
2. `KMFA/metadata/mgmt-monthly-report-skill/` 存 public-safe metadata、hash、manifest、日志摘要、验收摘要和数据库 SQL 导出。
3. 原始敏感 Excel、合同、银行、税务、工资、token、明文报告正文和 SQLite runtime DB 不提交 GitHub。
4. 本机运行目录最终只保留正式 `.xlsx` 和 `.pdf` 报告；其他 skill-created cache/temp/log/database 可以清理。
5. 不自动删除用户原始文件；删除或移动原始文件需要当前线程单独明确授权。

## 验收规则

1. `validate_skill_package.py` 必须通过。
2. 正式报告生成后，`validate_deliverables.py --strict` 必须通过。
3. 禁止词、乱码、公式错误、`#####`、默认 Series 图例均为失败项。
4. 验收失败时只输出问题清单，不得声称正式完成。
