# 02_数据源契约｜6 个业务输入文件

## 1. 输入原则

每期用户只需要提供 6 个业务输入包。Agent 必须按业务含义自动识别文件，不得依赖完全一致的物理文件名；但识别不出来时必须停止并要求用户补齐或确认映射。

## 2. 六个必需业务输入包

| 序号 | 业务输入包 | 典型文件名模式 | 必需 Sheet / 关键内容 | 缺失处理 |
|---:|---|---|---|---|
| 1 | WPS 武汉开明 2026年回款表 | `*2026*回款表*.xlsx`、`*2026*回款*.xlsx` | 回款日期、客户、合同号、回款金额、回款方式、负责人/备注等 | 缺失则停止 |
| 2 | 开票纳税资金汇总表 | `*开票*纳税*资金汇总*.xlsx` | `开票纳税汇总`、`2026年销售回款`、`2026年资金汇总` | 缺失则停止；旧版 `.xls` 必须先转换为 `.xlsx` |
| 3 | 应收账款合同登记 | `*应收账款*合同登记*.xlsx`、`*合同登记*.xlsx` | 合同客户、合同金额、已收、剩余、合同状态 | 缺失则停止 |
| 4 | 应收账龄表 | `*应收账龄*.xlsx` | 账龄、客户、金额、开票、回款、未回款 | 缺失则停止 |
| 5 | 2026年保证金 | `*保证金2026*.xlsx`、`*保证金*.xlsx` | 支付类型、支付金额、退款、未退、公司、日期 | 缺失则停止 |
| 6 | 项目合同状态包 | `*三大项目*.xlsx`、`*生产项目状态表*.xlsx`、`*红圈主合同*.xlsx` | 项目、合同、开工、完工、结算、开票、回款、主合同信息 | 缺失则停止；若被拆成多个物理文件，仍归为一个业务包 |

## 3. 固定参考文件

以下文件不是每期必须变化，但如果用户提供，必须利用；如果没有提供，必须从 v6 基准或既有历史规则中读取静态口径，并在 `10_口径说明与分析方向` 中写明：

| 文件 | 用途 |
|---|---|
| `主营业务描述.doc/docx` | 经营分析的业务背景、行业场景、技术服务能力 |
| `资质及三类人员情况.docx` | 资质、人力、三类人员、特种设备人员要求 |
| `花名册.xlsx` | 组织与产能画像、人员结构 |

公司业务背景不能当成普通贸易业务处理：主营业务资料显示，公司专门从事机械设备金属表面修复，具备 FGM-KM-B.U 新材料/新工艺、鄂州生产基地、大型加工设备、100T 起重机/加热炉、24H 应急抢修等能力。因此报表必须串联合同、项目、完工、开票、回款、应收、保证金、资质人力和产能。资质文件也显示，武汉开明涉及电力、石化、防腐、机电、环保、GC2、GB1/GB2、锅炉 B 级等资质，以及三类人员和特种设备人员社保要求。

## 4. 文件识别规则

Agent 执行时必须先生成输入映射表：

| 字段 | 含义 |
|---|---|
| business_group_id | 业务输入包 ID |
| expected_patterns | 期望文件名 |
| matched_files | 实际匹配文件 |
| required_sheets | 必需 Sheet |
| sheet_check_result | Sheet 检查结果 |
| status | 通过 / 缺失 / 冲突 / 需要确认 |

任何必需业务包缺失，正式 Excel/PDF 不得生成。

## 5. `.xls` 文件处理

- 若输入为 `.xls`，Agent 不得直接假设能稳定解析。
- 必须先转换为 `.xlsx` 或要求用户提供 `.xlsx`。
- 如果转换后格式与源表不一致，必须停止并报告。

## 6. 资金与开票源表硬规则

`02_开票纳税汇总`、`03_2026年销售回款`、`04_2026年资金汇总` 只能使用 `开票纳税资金汇总表` 中指定 Sheet。不得从其他文件拼凑、猜测或替代。

- `02_开票纳税汇总` 只允许使用 Sheet `开票纳税汇总`。
- `03_2026年销售回款` 只允许使用 Sheet `2026年销售回款`。
- `04_2026年资金汇总` 只允许使用 Sheet `2026年资金汇总`。
