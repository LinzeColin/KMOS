# 经营管理分析报表任务包 202607｜README

## 结论
本任务包用于把本期 6 个业务原始输入包，稳定生产为 v6 基准样式的两个交付物：

1. `经营管理分析报表 YYYYMM.xlsx`
2. `董事会经营分析摘要 YYYYMM.pdf`

本任务包的目标不是让 Agent 自由发挥，而是让任意 LLM / Agent / Codex 严格复现 v6 基准版的信息架构、表格顺序、字体、颜色、行高、列宽、图表、口径、公式和校验流程。

## 任务包内容

| 序号 | 文件 | 用途 |
|---:|---|---|
| 01 | `01_模板基准规范.md` | 固化 v6 的 Sheet 顺序、视觉规范、图表规范、命名规范 |
| 02 | `02_数据源契约_6个输入文件.md` | 定义每期必须提供的 6 个业务输入包及缺失处理 |
| 03 | `03_指标口径与跨表耦合.md` | 固化回款、开票、合同、完工、应收、保证金、资金流等计算口径 |
| 04 | `04_Excel生成规范.md` | 规定 Excel 生成细节：布局、公式、图表、格式、禁止事项 |
| 05 | `05_PDF董事会摘要规范.md` | 规定 PDF 董事会摘要内容、页数、口径、视觉和风险表达 |
| 06 | `06_Recurring_Prompt_经营管理分析报表.md` | 给任意 LLM / Agent 的完整执行提示词 |
| 07 | `07_Codex_Task_Pack_周度月度执行.md` | 给 Codex 的 read-only → plan → execute → verify 工作包 |
| 08 | `08_自动验收规则.md` | 输出前必须执行的验收项，禁止凭肉眼或主观判断通过 |
| 09 | `09_版本变更记录与防回退.md` | 防止后续版本回退、遗忘、重构污染和样式漂移 |
| config | `v6_spec.json` | 机器可读规范：Sheet 顺序、标签颜色、禁用词、图表要求 |
| config | `input_manifest.template.yml` | 每期输入文件清单模板 |
| scripts | `validate_deliverables.py` | 自动验收脚本：检查 Excel / PDF / 输入文件 / 样式 / 公式 / 图表 |
| golden | `经营管理分析报表 v6基准.xlsx` | 当前已验收的 v6 基准文件，后续必须以此为样式和结构基准 |

## 执行方式

```bash
python scripts/validate_deliverables.py   --period 202607   --input-dir ./inputs   --excel "./outputs/经营管理分析报表 202607.xlsx"   --pdf "./outputs/董事会经营分析摘要 202607.pdf"   --config ./config/v6_spec.json   --report "./outputs/自动验收报告 202607.json"   --strict
```

## 核心原则

- 必须从 v6 基准继续，不得从空白文件重建样式。
- 必须用用户真实数据，不得使用测试数据、示例数据、估算数据、占位数据。
- 必须公式驱动，不得把资金流入、资金流出、累计资金净流量等关键结果硬编码为值。
- 必须保留 Excel 原生可编辑图表，不得把图表截图粘贴成不可编辑图片。
- 必须输出 `FileName YYYYMM` 命名格式，不得输出带日期横杠、随机版本号或 URL 编码文件名。
- 必须在输出前运行自动验收；失败时不得交付正式文件，只能交付问题清单。
