# KMDatabase

KMFA / KMIDS / whkmSalary 共享的数据与元数据层。

## 为什么存在

三个 KM 项目拆分为独立目录后，仍需交换数据。若无共享契约层，
跨项目引用会退化为直接引用对方内部路径 —— 脆弱、无法版本化、
一方重构就会静默打破另一方。

KMDatabase 是这些交换的**单一事实源**。

## 结构

- `schema/` —— 跨项目数据结构定义。任何跨项目交换的数据都必须在此声明。
- `data/` —— 实际数据。**不入库**（见 .gitignore），仅存在于本地与备份。

## 约定

- 修改 `schema/` 下任何定义，必须同步确认三个消费方均已适配。
- 不要把单个项目私有的数据结构放进来 —— 那属于该项目自己。
